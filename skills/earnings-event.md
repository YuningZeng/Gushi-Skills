# 事件与财报跟踪 Skill

## 目标
在财报季、政策变化、行业拐点和公司重大事项发生前后，帮助投资者判断：公司长期价值是否改变，是否需要调整仓位或预期。

## 适用场景
- 年报 / 中报 / 季报发布前后
- 政策出台前后
- 公司重大经营事件发生时
- 行业出现景气变化时

## 输入信息
- 公司名称 / 股票代码
- 事件类型
- 事件日期
- 当前持仓
- 市场预期
- 历史估值位置
- 关键财务或经营指标
- 自己的核心投资逻辑

## 分析流程
1. **识别事件重要性**
   - 这是短期扰动，还是长期逻辑变化？
2. **对照原有投资逻辑**
   - 事件是否强化了逻辑，削弱了逻辑，还是与逻辑无关？
3. **检查预期差**
   - 市场反应是否过度？
   - 价格变化是否已经反映了事件影响？
4. **形成动作建议**
   - 继续观察
   - 维持持有
   - 分批加仓
   - 适度减仓
   - 退出
5. **列出后续验证点**
   - 下一季报要验证什么？
   - 管理层表态需要跟踪什么？

## 输出格式
- 事件性质判断
- 对原逻辑的影响
- 市场预期差
- 建议动作
- 后续验证点

## 核心规则
- 财报和事件不只看股价反应，更要看逻辑是否变了
- 一次超预期不等于长期趋势成立
- 一次低于预期，也不一定立刻破坏长期价值
