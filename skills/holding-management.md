# 持仓管理 Skill

## 目标
帮助投资者在持有过程中，用基本面和估值框架决定“继续持有、分批加仓、适度减仓、还是退出”，而不是被短期涨跌牵着走。

## 适用场景
- 股价大涨后是否减仓
- 股价大跌后是否继续持有或加仓
- 定期检查核心持仓
- 发现逻辑变化后处理仓位

## 输入信息
- 公司名称 / 股票代码
- 初始投资逻辑
- 当前股价与成本价
- 当前估值水平
- 最新财报与经营变化
- 当前仓位比例
- 组合中的定位
- 是否出现新的重大风险

## 处理流程
1. **先看逻辑是否还成立**
   - 公司长期价值是否发生变化？
2. **再看估值是否变化**
   - 现在是更有吸引力了，还是已经明显高估？
3. **判断四种动作**
   - 继续持有
   - 分批加仓
   - 适度减仓
   - 完全退出
4. **加仓条件**
   - 基本面未恶化
   - 估值更具吸引力
   - 组合权重仍可控
5. **减仓条件**
   - 估值明显透支
   - 逻辑边际变弱
   - 组合过于集中
6. **退出条件**
   - 投资逻辑失效
   - 财务质量恶化
   - 管理层或治理出现重大问题

## 输出格式
- 当前逻辑状态
- 估值状态
- 持仓定位
- 推荐动作
- 后续跟踪点
- 风险提醒

## 核心规则
- 不因为短期下跌就轻易否定长期逻辑
- 不因为短期上涨就忘记估值约束
- 持仓管理的本质是动态权衡“价值、价格、风险”
